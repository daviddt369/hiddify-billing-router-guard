# Политика безопасности

## Где хранятся секреты

Все секреты хранятся вне репозитория:

| Секрет | Расположение |
|---|---|
| Токен Telegram-бота | Панель администратора → Business → Telegram (хранится в БД панели) |
| Токен провайдера YooKassa / Telegram Payments | Панель администратора → Business → YooKassa (хранится в БД панели) |
| Webhook-секрет Telegram | `/etc/hiddify-panel/panel-secrets.env` (генерируется при установке командой `openssl rand -hex 32`) |
| Секретный ключ приложения | `/opt/hiddify-manager/hiddify-panel/app.cfg` |

Ни одно из этих значений не хранится в репозитории. Webhook-секрет генерируется установщиком автоматически во время установки.

## Что нельзя коммитить

Следующие данные никогда не должны попадать ни в один коммит, ветку или тег этого репозитория:

- Токены Telegram-ботов (шаблон: `<цифры>:<буквенно-цифровая строка>`)
- Токены YooKassa или других провайдеров платёжных систем
- Приватные ключи любого типа (`BEGIN PRIVATE KEY`, `BEGIN RSA PRIVATE KEY` и т. д.)
- Реальные IP-адреса production-серверов
- Реальные доменные имена production-серверов или внешних нод
- Имена пользователей, hostname серверов или SSH-ключи
- Пароли к базам данных или строки подключения с учётными данными
- Любые файлы вида: `.env`, `*.sql`, `*.log`, `*secret*`, `panel-secrets.env`

Эти шаблоны контролируются через `.gitignore`. Проверяйте перед каждым push:

```bash
git grep -E '[0-9]{8,12}:[A-Za-z0-9_-]{30,45}'
git grep -E 'BEGIN (RSA |EC |OPENSSH )?PRIVATE KEY'
```

## Обработка токена Telegram

Токен Telegram-бота загружается во время выполнения из базы данных панели через `hconfig(ConfigEnum.telegram_bot_token)`. Токен никогда не записывается на диск кодом приложения и не появляется в логах.

Если в пакете telegrambot присутствует модуль `secrets.py` (не включённый в этот репозиторий), он имеет приоритет над значением из базы данных. Это позволяет использовать инъекцию секретов через окружение без доступа к БД.

## Требование к webhook-секрету

Endpoint webhook (`/api/v2/tgbot/`) работает по принципу fail-closed: если переменная окружения `HIDDIFY_TELEGRAM_WEBHOOK_SECRET` или соответствующая запись в `/etc/hiddify-panel/panel-secrets.env` отсутствует или пуста, **все входящие запросы webhook отклоняются с HTTP 403**. Секрет проверяется с помощью `hmac.compare_digest` для защиты от timing-атак.

Установщик создаёт секрет автоматически. Для ротации секрета:

1. Сгенерируйте новый секрет: `openssl rand -hex 32`
2. Обновите `/etc/hiddify-panel/panel-secrets.env`
3. Повторно зарегистрируйте webhook: перейдите в Business → Telegram в панели администратора и сохраните настройки

## Сообщение об уязвимостях

Если вы обнаружили уязвимость безопасности:

1. Создайте GitHub Issue в этом репозитории с общим описанием уязвимости.
2. Не публикуйте токены, ключи, адреса серверов или детали эксплойта публично.
3. Если проблема конфиденциальна, запросите приватное обсуждение через GitHub Issue.

## Чеклист безопасности перед релизом

Перед созданием любого тега:

- [ ] `git grep -E '[0-9]{8,12}:[A-Za-z0-9_-]{30,45}'` — нет токенов Telegram
- [ ] `git grep -E '[0-9]{5,}:(TEST|LIVE):[A-Za-z0-9_:-]+'` — нет токенов YooKassa
- [ ] `git grep 'BEGIN PRIVATE KEY'` — нет приватных ключей
- [ ] Нет реальных IP-адресов и hostname серверов ни в одном отслеживаемом файле
- [ ] Нет реальных доменных имён в документации (используйте плейсхолдеры `<DOMAIN>`)
- [ ] `.gitignore` покрывает `*.sql`, `*.log`, `.env`, `*secret*`, `*.bak`
- [ ] `.gitattributes` задаёт LF-окончания строк для всех текстовых файлов
- [ ] Все `.sh`-файлы проходят проверку `bash -n`
- [ ] Все `.py`-файлы проходят проверку `python3 -m py_compile`
