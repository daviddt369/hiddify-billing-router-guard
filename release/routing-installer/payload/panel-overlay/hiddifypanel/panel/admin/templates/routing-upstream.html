Как работают внешние ноды:
xray-router получает трафик от основного Hiddify (через SOCKS5 127.0.0.1:20808) и направляет его:
— RU-трафик (домены .ru/.su, geoip:ru, ваши правила) → direct-ru (напрямую из РФ)
— Остальной трафик → первая включённая нода по приоритету (наименьший номер = активная нода)
Нода применяется только после нажатия «Применить» в разделе «Маршрутизация».

Что такое внешняя нода (upstream)?
Это сервер за пределами РФ, через который xray-router пропускает глобальный трафик. Поддерживаются VLESS, Trojan и WireGuard. «Тестовый blackhole» означает, что трафик будет сброшен — используется только для проверки работы роутера без реальной ноды.
Идентификация ноды
Уникальный идентификатор: только латиница, цифры, дефис, подчёркивание. Не меняйте после создания.
Произвольное описание для вашего удобства.
0 = наивысший приоритет. Первая включённая нода с наименьшим числом становится активной.
Подключение к внешней ноде
VLESS и Trojan подходят для существующих прокси-серверов. WireGuard — для VPN-туннеля. Blackhole используйте только для тестирования маршрутизации без реального выхода.
{# VLESS #}
Скопируйте VLESS URI из вашего прокси-сервера (например из Marzban, 3x-ui, и т.д.). Формат: vless://UUID@host:port?параметры
{# Trojan #}
Формат: trojan://пароль@host:port?параметры. Обычно берётся из настроек вашего Trojan-сервера.
{# WireGuard #}
Адрес WireGuard-сервера, например: 1.2.3.4:51820
Публичный ключ WireGuard-сервера (из конфига сервера, поле PublicKey пира).
Можно указать: сам ключ в base64, env:ИМЯ_ПЕРЕМЕННОЙ или file:/путь/к/файлу. Приватный ключ клиента (ваш ключ, не серверный).
IP-адрес клиента в WireGuard-сети. Обычно назначается сервером.
Обычно 1280 или 1420. Уменьшите если возникают проблемы.
