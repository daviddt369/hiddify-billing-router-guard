# Примечания к релизу — v1.0.0-rc1

**Дата релиза:** 2026-05-17
**Статус:** Release Candidate — функциональное тестирование завершено; перед развёртыванием на production рекомендуется провести ревью.

---

## Краткое описание

v1.0.0-rc1 — первый публичный релиз-кандидат Hiddify Commercial Addon Stack. Поставляет все три аддона (business, routing, antishare) как единый стек с атомарной установкой и поддержкой отката для HiddifyPanel 12.0.0.

---

## Что нового

### Установщик

- `release/clean-install-full-stack.sh` — устанавливает все три аддона последовательно (business → routing → antishare) с preflight-проверками и smoke-тестами на каждом этапе.
- Ожидание порта 9000 — установщик ждёт до 120 секунд, пока панель не начнёт слушать порт 9000, предотвращая гонки на медленных серверах.
- Автоматическая регистрация webhook — после установки business-аддона webhook регистрируется автоматически, если токен бота уже настроен.
- Атомарный откат — каждый установщик сохраняет резервные копии файлов на уровне файловой системы перед любыми изменениями; при ошибке файлы восстанавливаются в обратном порядке.

### Telegram-бот

- Динамическая reply-клавиатура — кнопки адаптируются в зависимости от наличия активного тарифа у пользователя.
- Инструкции по платформам — inline-клавиатура позволяет выбрать Android, iPhone или Windows; текст инструкций настраивается из панели администратора.
- Автоматическая выдача пробного доступа — при отправке пользователем номера телефона аккаунт создаётся автоматически с настраиваемыми лимитами.
- Ссылка на подписку в статусе — URL подписки вставляется в сообщение о статусе для удобного копирования в VPN-клиент.
- Напоминания об истечении — настраиваемые напоминания отправляются автоматически через Celery-задачу за указанное количество дней до окончания подписки.
- Уведомления администраторов — администраторы получают уведомления в Telegram о новых регистрациях, запросах тарифов и событиях оплаты.

### Миграции базы данных

- `_v137` — добавляет ключи конфигурации инструкций по платформам (`telegram_instruction_android`, `telegram_instruction_ios`, `telegram_instruction_windows`) в таблицу `str_config`.
- `_v138` — добавляет `ON DELETE CASCADE` к внешнему ключу `user_detail`, предотвращая появление «осиротевших» записей при удалении пользователей.

### Исправления ошибок

- Падение `_v136` без routing-аддона — `BusinessAdmin.py` теперь корректно обрабатывает случай, когда routing-аддон не установлен; раздел маршрутизации скрывается из UI вместо возврата ошибки 500.
- Ссылка на подписку вне request-контекста — `_send_my_subscription` теперь строит URL подписки через тестовый request-контекст, когда вызывается вне HTTP-запроса (например, из Celery-задачи), предотвращая `RuntimeError: Working outside of application context`.
- Ошибка 500 webhook — удалён сегмент UUID из URL tgbot, чтобы не активировать auth-middleware, возвращавший 500 на preflight-запросах.

### Безопасность

- Автоматическое создание webhook-секрета — установщик вызывает `openssl rand -hex 32` и записывает результат в `/etc/hiddify-panel/panel-secrets.env` перед регистрацией webhook.
- Fail-closed валидация webhook — если webhook-секрет отсутствует или пуст, все POST-запросы на `/api/v2/tgbot/` отклоняются с HTTP 403.
- Дедупликация обновлений — скользящее окно дедупликации длиной 15 минут предотвращает двойную обработку одного `update_id`.

## Ожидаемые предупреждения (не ошибки)

- **"Telegram bot token is not configured"** — записывается в лог при запуске панели до настройки токена в панели администратора. Штатное поведение.
- **xray-router inactive / upstream unreachable** — записывается health probe маршрутизации, пока не настроена хотя бы одна внешняя нода. Штатное поведение.

## Инструкция по обновлению

Для обновления business-слоя после обновления базовой панели HiddifyPanel:

```bash
sudo bash release/upgrade-installer/upgrade-business-layer.sh
```

Этот скрипт повторно накладывает файлы business-аддона поверх обновлённой панели без изменения routing и antishare. После этого запустите smoke-тесты для routing и antishare.

---

## Откат

```bash
sudo bash release/rollback-all.sh
```

Или по отдельности в обратном порядке:

```bash
sudo bash release/antishare-installer/rollback-antishare.sh
sudo bash release/routing-installer/rollback-routing.sh
sudo bash release/business-installer/rollback-business.sh
```

Каждый скрипт отката восстанавливает файлы из последнего backup-каталога и опционально восстанавливает базу данных из SQL-дампа, сохранённого при установке.
